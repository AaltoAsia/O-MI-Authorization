Rules

Set rules for group/role

POST /set-rules

Set permission rules for a group/role. These are seperated in allow and deny rules. In O-MI Node the rules are prosessed by first filtering the incoming request by allow rules and then filtering with the deny rules from the result.

To set rules for consumer, it either needs to be added to groups or the automatic username_USERGROUP can be used.

SetRules

Parameters in json

Request Example
{
  "group": "string",
  "rules": [
    {
      "path": "string",
      "allow": "boolean",
      "request": "string"
    }
  ]
}

Get rules of a consumer and a request type

POST /get-permissions

Permissions of a consumer for a request type. This operation should be used by O-MI Node for authorization, meaning the result is compatible with Authorization API service v2.

If the consumer has connections to many groups/roles (with join-groups) and they have many rules, the rules are concatenated in the following way:

  • allow rules are combined with set union
  • deny rules are combined with set intersection

Hierarchical structure of the O-DF paths is taken into account when doing these set operations. This way the consumer, belonging to a certain group, can not lose any permissions in that group just by joining other groups.

Parameters in json.

  "request": "string"
}

Schema Definitions

Request: string

Select one or more requests:

  • r for read request
  • w for write request
  • c for call request
  • d for delete request

When setting new rules, many request types can be used by concatenating them.

string r, w, c, d
